02

PHYS

 

第一關：物理量測

song_scorer.py · librosa + pyloudnorm + parselmouth

程式無法「聽懂」音樂，但能量測聲音的物理特徵。先自動偵測 調性（Krumhansl-Kessler chroma correlation） 和 BPM（librosa beat_track + 拍距中位數外插網格） 作為內部基準，再對混音進行 7 項客觀量測，各項 0–100 分後加權。

  整體響度 （15%）

pyloudnorm 量測 integrated LUFS。以 -14 LUFS（串流標準）為高分中心，偏離越遠越低。LUFS > -13.5 建議正規化，-14.6 ~ -13.5 標示「發行就緒」。

  動態範圍 （20%）

峰值／均方根比 (crest factor)。4–16 dB 之間最佳，過低表示壓縮過重，過高表示動態失控。

  頻譜平衡 （20%）

STFT 能量分四區：低 60–250Hz、中低 250–2k、中高 2k–6k、空氣 6k+。每區定義合理佔比範圍，偏離越多 penalty 越大，公式為 clip(100 − 250 × penalty, 30, 100)。

  立體聲寬度 （10%）

以 (L−R)/(L+R) 的 RMS 比計算寬度指數，輔以左右相關係數。指數 0.2–0.7 最佳，過低趨近單聲道，過高有反相風險。

  削波檢測 （10%）

樣本點觸頂（>0.999）比例。0% 為滿分，超過 0.1% 即大幅扣分。

  層次鋪陳 （15%）

切 8 段量 RMS 落差與頻譜質心變異係數，加權 6：4。能量落差 6–15 dB 配合音色變化，分數最高。

  和聲豐富度 （10%）

諧波分離後 chroma CQT 計算所用音級數（6–9 最佳）與半小節解析度的和弦變化率（0.15–2.0 次/秒最佳），兩者平均。

總分 = Σ(score_i × weight_i) / Σ(weight_i) 缺項自動剔除並重新正規化權重。等級：S ≥ 90、A ≥ 80、B ≥ 70、C ≥ 60、D < 60。演唱分析為選配（需人聲軌或 demucs 分離），含音準、節奏、顫音、音域等 7 項。

03

AES

 

第二關：美學模型

SongEval + Audiobox Aesthetics

SONGEVAL

音樂人訓練模型 · 1–5 分

複製自 ASLP-lab/SongEval （CC BY-NC-SA，Space 啟動時 git clone）。五維度各有 1–5 分，取平均後給等級判定：優秀 ≥4.25、良好 ≥3.5、中等 ≥2.75。任一維度低於 4 時特別標註。

整體連貫性 — 段落銜接、整體流暢度； 整體音樂性 — 旋律與編曲的音樂質感； 記憶點 — hook 的記憶程度； 結構清晰度 — 主副歌結構； 人聲自然度 — 聽感自然程度。

AUDIOBOX

Meta 美學模型 · 1–10 分

audiobox-aesthetics v0.0.4，四軸評分。其中 CE（內容感染力） 若低於 7 分，自動觸發親聽檢查警報，提醒使用者戴上耳機確認情緒傳達。

PQ 混音清晰度・ CE 感染力・ CU 場景適配・ PC 編曲密度（描述性，非越高越好）。

04

EMO

 

中介：情感弧線

情感弧線.py · NRC-VAD Russell 模型

以 Russell (1980) 情感環狀模型 為理論基礎，Valence（愉悅度）× Arousal（激動度）雙軸 0–1 制。詞庫使用加拿大國家研究院的 NRC-VAD Lexicon，中英雙語自動偵測：含中文字元用中文詞庫（簡+繁合併、最長匹配掃描），否則用英文原生詞庫。

歌詞.txt → 段落分割 → NRC-VAD 查表 → 逐段 V/A 均值 → 雙圖 PNG 輸出三段式分析（最負面詞 Top 3、最激動詞 Top 3）與 matplotlib 雙圖：上圖為 V/A 隨段落折線（含 0.5 中性線），下圖為 Russell 平面路徑。擺幅 ≥0.05 判定為有意義情緒移動。數據包裝成 _measure_block() 餵給詞評 LLM 做合議庭裁決的交叉引用佐證。

  Prompt 結構

「_lyric_prompt()」封裝約 1.5K 字的評詞標準（LEAN_RUBRIC）與實際量測數據，組成 4–8K tokens 的完整 prompt，max_tokens=5000。依序要求 LLM 輸出：題材定尺、七維度雙分數表格（作品分 + 爆款分）、情感三支柱、傳播假設檢查表、句級修法、場景適配表、合議庭裁決，最後強制附上 【詞總分】X.X 供排行榜排序。

06

RANK

 

排行榜計分

作品發行視角加權 · 最多 100 名

Total = 詞分×10×0.4 + (SongEval平均÷5×100)×0.3 + CE×10×0.3

詞佔 40%、SongEval 美學佔 30%、CE 感染力佔 30%。 物理分獨立顯示為「發行就緒度」 ，不混入總分——因為後製可抹平，不應影響歌曲本身評價。前端只持有 audio_hash，分數由伺服器自 JSON 快取重算，無法偽造。同名自動加 v2/v3 去重。LLM 漏寫詞總分行時，自動從七維度表格平均值補。

SUMMARY

song_scorer.py 的 7 項物理量測 ＋ SongEval 五維 ＋ Audiobox 四軸 ＋ NRC-VAD 情感弧線 ＋ DeepSeek→Groq 詞評，構成一套從物理到文本的全自動歌曲評審管線。快取層以雙指紋索引聰明重評，排行榜以前端不可偽造的方式呈現作品發行視角下的綜合排名。核心設計目標：在零付費 API 額度的前提下，提供有客觀數據錨點的創作診斷。
