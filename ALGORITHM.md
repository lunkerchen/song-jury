## 第一關：物理量測 — `song_scorer.py`

**本質**：程式無法「聽懂」音樂，但能量測聲音的物理特徵。

### 步驟流程

```
音檔 → 載入(librosa) → 調性偵測 → 節拍偵測 → 7 項量測 → 加權總分
```

### 基準偵測（前置）

| 項目 | 方法 | 輸出 |
|---|---|---|
| **調性 (Key)** | Chroma CQT → Krumhansl-Kessler 12-bin profile 相關係數（掃 12 種移調 × 大/小調） | C 大調 / A 小調… + 信心值 |
| **BPM** | `librosa.beat.beat_track` → 拍距中位數 → 全曲外插網格 | BPM + 拍點時間陣列 |

### 7 項混音量測

每項 0–100 分，用 `piecewise()` 分段線性函數將量測值映射到分數。

| 維度 | 權重 | 量測方法 | 評分曲線（量測值→分數） |
|---|---|---|---|
| **整體響度** | 15% | `pyloudnorm` integrated LUFS | -30→40, -22→70, -18→90, -16→100, -9→100, -7→85, -4→60 |
| **動態範圍** | 20% | 峰值/均方根比 (crest factor) dB | 4→40, 6→65, 8→85, 10→100, 16→100, 20→85, 26→65 |
| **頻譜平衡** | 20% | STFT 能量：低(60–250Hz)/中低(250–2k)/中高(2k–6k)/空氣(6k+)各佔比 | `clip(100 - 250 × penalty, 30, 100)`，penalty = 各區偏離合理範圍量 |
| **立體聲寬度** | 10% | `(L-R)/(L+R)` RMS 比 + 左右相關係數 | 0.02→55, 0.1→75, 0.2→90, 0.35→100, 0.7→100, 0.9→85, 1.2→65 |
| **削波檢測** | 10% | 樣本觸頂（`>0.999`）比例 | 0→100, 0.0001→95, 0.001→75, 0.01→40, 0.05→15 |
| **層次鋪陳** | 15% | 切 8 段，量 RMS 落差 + 頻譜質心變異係數 | 能量落差 + 音色變化加權 6:4 |
| **和聲豐富度** | 10% | 諧波分離 → chroma CQT → 音級數 + 半小節解析度和弦變化率 | 音級數 + 變化率平均 |

**總分公式**：
```
total = Σ(score_i × weight_i) / Σ(weight_i)   // 缺項自動剔除重正規化
grade =  S(≥90) / A(≥80) / B(≥70) / C(≥60) / D
```

**響度發行建議**（三檔）：
- LUFS > -13.5 → 「建議正規化到 -14 LUFS」
- -14.6 ~ -13.5 → 「已在標準附近，發行就緒」
- < -14.6 → 「略低於標準」

### 演唱分析（選配 — 需人聲軌或 demucs）

人聲軌 → pyin 基頻抽取 → 音符分段 → 7 項分析：

| 維度 | 方法 |
|---|---|
| **音準** | 每音符中位數距最近半音的音分偏差 + 調內音符比例（8:2 加權） |
| **節奏** | 音符起點對齊節拍網格 + 八分音符細分，量殘差一致性 |
| **長音穩定** | 無顫音長音的音分 MAD（中位數絕對偏差） |
| **顫音** | FFT 偵測 3.5–8.5 Hz 調製，量速率/深度/出現率 |
| **動態控制** | 有聲段 RMS 的 90–10 百分位距 |
| **嗓音品質** | Praat parselmouth jitter/shimmer/HNR |
| **音域** | 音符中位數的 5–95 百分位跨度 |

## 第二關 A：SongEval（音樂人美學模型）

**來源**：ASLP-lab/SongEval（CC BY-NC-SA，Space 啟動時 `git clone --depth 1`）

**五維度**（各 1–5 分）：

| 維度 | 英文 | 說明 |
|---|---|---|
| 整體連貫性 | Coherence | 段落銜接、整體流暢度 |
| 整體音樂性 | Musicality | 旋律與編曲的音樂質感 |
| 記憶點 | Memorability | hook／旋律讓人記住的程度 |
| 結構清晰度 | Clarity | 主副歌結構是否清楚 |
| 人聲自然度 | Naturalness | 人聲聽感的自然程度 |

輸出為平均分 + 等級判斷（≥4.25=優秀／≥3.5=良好／≥2.75=中等…），並標註最低維度是否低於 4。

## 第二關 B：Audiobox Aesthetics（Meta 美學模型）

**套件**：`audiobox-aesthetics==0.0.4`

**四軸**（各 1–10 分）：

| 軸 | 說明 |
|---|---|
| **PQ** 製作品質 | 混音/清晰度的技術水準 |
| **CE** 內容感染力 | 情緒感染與聆聽愉悅度 |
| **CU** 內容實用性 | 場景適配的廣度 |
| **PC** 製作複雜度 | 編曲密度（描述性，非越高越好） |

若 CE < 7 → 觸發「親聽檢查」警報。

## 中介：情感弧線 — `情感弧線.py`

### 理論基礎

Russell (1980) 情感環狀模型：**Valence（愉悅度）** × **Arousal（激動度）**，0–1 制，0.5 = 中性。

### 詞庫

**NRC-VAD Lexicon**（Mohammad 2018，加拿大國家研究院，研究用途免費）：
- 中文：簡+繁合併，最長匹配掃描
- 英文：斷詞後查原生詞庫

雙語自動偵測：歌詞含中文字元 → 用中文詞庫；否則 → 英文詞庫。

### 流程

```
歌詞.txt → [標記]或空行分段 → 逐段查 V/A 值 → 取平均值
    → 輸出三段式分析 + matplotlib 雙圖
```

### 輸出

1. **主控台**：逐段 V/A、最負面詞 Top 3、最激動詞 Top 3
2. **弧線判定**：首尾位移 + 最大擺幅（≥0.05=有意義移動）
3. **雙圖 PNG**：
   - 上圖：V/A 隨段落折線 + 0.5 中性參考線
   - 下圖：Russell 平面路徑（附號碼→段落名對照）

### LLM 交叉引用

物理層和美學層的數據被包裝成 `_measure_block()` 字串，注入詞評 prompt 供合議庭裁決交叉引用。

## 第三關：AI 詞評 — LLM 多供應商鏈

### 供應商鏈（品質優先，逐家 fallback）

| 順位 | 供應商 | 模型 | 特性 | 時限 |
|---|---|---|---|---|
| 1 | DeepSeek | deepseek-v4-pro | 671B，中文母語最強 | 240s |
| 2 | Groq | openai/gpt-oss-120b | 免費、快、會交叉引用數字 | 90s |
| 3 | Groq | llama-3.3-70b-versatile | 免費、最穩的底 | 90s |

沒設金鑰 → 自動跳過該家。回應太短（<300 字）→ 換下一家。每家有 1 次重試（429/500 容錯）。

### Prompt 結構（`_lyric_prompt()` 封裝）

1. **題材定尺**：嘻哈/民謠/搖滾/古風/流行 — 決定評判尺
2. **七維度雙分數表格**（各 0–10）：主題立意、敘事結構、意象畫面、音韻押韻、歌唱性、聲腔與新鮮度、金句記憶點 → 每維度給「作品分」+「爆款分」，需引原句
3. **情感三支柱**：客觀對應物、張力管理、情感弧線判讀
4. **傳播假設檢查表**：hook 30 秒內、可剪句、二創場景、受眾語境
5. **句級修法**：3 條以上的問題→建議→改寫
6. **場景適配表**：作品發行/短影音/舞曲/品牌配樂
7. **合議庭裁決**：交叉引用物理+美學數據，給發行建議+必修三步
8. **詞總分**：強制 `【詞總分】X.X`，供排行榜排序用

### 規模

評詞標準全文（`LEAN_RUBRIC`）約 1.5K 字。整份 prompt 含歌詞約 4–8K tokens。
max_tokens = 5000 給足中文輸出空間。

## 排行榜計分

### 上榜資格

需有完整詞評分 + SongEval 平均 + Audiobox CE。

### 作品發行視角加權

```
Total = 詞分×10×0.4 + (SongEval平均÷5×100)×0.3 + CE×10×0.3
```

- 詞 40% + SongEval（美學）30% + CE（感染力）30%
- 物理分獨立顯示為「發行就緒度」，不混入總分
- 最多 100 名
- 同名自動加 v2/v3 （去重）

### 安全機制

- 前端只持有 `audio_hash`，分數由伺服器自 JSON 快取重算
- 排行榜寫入有衝突重試（最多 4 次）
- 詞分解析有後備：LLM 漏寫 `【詞總分】X.X` 時，自動從七維度表格平均值補

## 快取與聰明重評

### 三張快取表（私有 dataset）

| 表 | 索引 | 用途 |
|---|---|---|
| `reports.json` | audio_hash（音檔 SHA256 前 16 碼） | 音訊三關結果 + 裁決集 |
| `lyrics.json` | lyrics_hash（去空白 SHA256 前 16 碼） | 詞評主體（版本化） |
| `sources.json` | 正規化 URL | 連結→audio_hash 映射 |

### 快取命中邏輯

| 狀態 | 行為 | API 花費 |
|---|---|---|
| 🟢 完全命中（同音檔×同詞） | 整份沿用 | 零 |
| 🟡 詞評命中、母帶換了 | 沿用詞評主體，僅重生「合議庭裁決」 | 小額 DeepSeek |
| 🔴 全新 | 完整三關 + 詞評 | 全額 |

版本控制：`ENGINE_VERSION`（e1）和 `RUBRIC_VERSION`（v8）任一 bump → 對應快取失效重評。

### 排行榜儲存（公開 dataset）

`leaderboard.json` 結構：
